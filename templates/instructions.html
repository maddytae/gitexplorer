Instructions:

 

Mode 1: Diff within a repo

 

In this mode, you can input repo address of any public repo. Currently public repos on GitHub, Bitbucket and Gitlab are supported. The address could be either SSH (for ex:- git@github.com:yaml/pyyaml.git) or HTTPS (for ex:- https://github.com/yaml/pyyaml.git). Once you submit the repo address you will be taken to repo page where you can do diff for both folders and files.

 

On the left side of the drop-down boxes, you can select a branch followed by the author and the commit hash. You can do same in the right side of the select boxes. Once these details are provided you will automatically get list of files that have been modified between the two commit hashes. You can select one of the files and perform usual file diff.

 

You also have option to do folder diff between the two commit hashes. There are two versions of the folder diff. The first one shows full folder tree which is union between the two commit hashes. The color code is as shown below: -

 

Unchanged: -             This is for any file that is common between the two commit hashes and the file content is unchanged between the two commits.

Deleted: -                    This is for files which are in left commit hash but are deleted in the right commit hash.

Added: -                      This is for files which are not in left commit hash but are added in the new commit hash.

Modified: -                   This is for files which are in both commit hashes. However, the file itself has been modified.

 

You can do file diff on all the categories above except for “Unchanged” since there is nothing to diff.

 

Mode 2: Diff outside Repo

 

This mode does not require any repo and is intended for some ad-hoc diff. There are three options for providing the input. You can directly paste your content inside input box or provide link to a file on web (for ex https://raw.githubusercontent.com/yaml/pyyaml/main/examples/yaml-highlight/yaml_hl.py) or upload a file. You can choose one of the three options on both left and right and they don’t need to be the same input method. For ex, you can upload a file on left side while you can directly paste content to diff on the right side and so on.

 

If there are multiple input methods i.e. both the input box and file upload are used, then the content inside input box will take precedence. The pecking order is input box, followed by file link and lastly the file upload.
